# 最終ゴール定義書

## 🚩 最終ゴール

### 1. 元素周期表から検索できるページを持つ
- 周期表をクリックして元素を選択
- 選択した元素を含む材料を検索・表示
- 元素カードと材料カードの連携

### 2. 118元素すべてを権利的に安全なデータでカード化
- IUPAC標準データ（パブリックドメイン）を使用
- 各元素の基本情報をカード形式で表示
- 元素記号、原子番号、原子量、周期、族、分類など

### 3. 材料カードに3タブ構造を持たせる
- **タブ1: 材料物性**
  - 既存の物性データ表示
  - 元素組成（将来実装）
- **タブ2: 入手先・用途**
  - 入手先情報
  - 用途情報
  - 使用例
- **タブ3: 歴史・物語**
  - 開発背景
  - 歴史的経緯
  - ストーリー

### 4. 将来的に「材料が元素周期表のどこに位置づくか」を視覚的にマッピング可能な構造を持つ
- 材料の元素組成から周期表上の位置を計算
- 周期表上に材料の位置をマッピング表示
- 複数元素を含む材料の可視化

## 📋 実装優先順位

1. **Phase 1**: 材料カードの3タブ構造（既存機能を壊さない）
2. **Phase 2**: 118元素のカード化（権利的に安全なデータ）
3. **Phase 3**: 元素周期表からの検索機能
4. **Phase 4**: 材料と元素の関連付け
5. **Phase 5**: 材料の元素周期表マッピング（将来実装）

## ✅ 実装原則

- 各ステップで「壊れない・動く」状態を維持
- 著作権・ライセンスに曖昧さがあるデータ・画像は使わない
- UI・データ・ロジックを分離
- 将来「材料が増える」「教育・研究用途に広がる」前提で設計
